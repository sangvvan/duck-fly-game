# Duck Fly Game - Development Team

## Team Structure

### 👑 Leader (You)
- **Role**: Project Lead & Coordinator
- **Responsibilities**:
  - Oversee overall project direction
  - Coordinate between team members
  - Make final decisions on features and design
  - Review team feedback and integrate improvements
  - Manage timeline and milestones

---

### 👨‍💻 Game Developer
- **Role**: Game Logic & Mechanics Development
- **Responsibilities**:
  - Implement game mechanics and logic
  - Optimize performance and game loop
  - Handle physics and collision systems
  - Review code for functionality
  - Suggest technical improvements

**Agent Task**: Analyzes code for game logic, performance, and architecture
**Runs**: On-demand or scheduled for code reviews

---

### 🎨 UI/UX Designer (iOS)
- **Role**: User Interface & Experience Design
- **Responsibilities**:
  - Design UI layouts and components
  - Ensure iOS design guidelines compliance
  - Create visual mockups
  - Optimize user interaction flow
  - Review designs for usability

**Agent Task**: Reviews UI/UX aspects, suggests design improvements
**Runs**: On-demand for design feedback

---

### 🧪 Tester
- **Role**: Quality Assurance & Testing
- **Responsibilities**:
  - Create test cases and test plans
  - Identify and document bugs
  - Test edge cases and game scenarios
  - Verify fixes
  - Performance testing

**Agent Task**: Generates test cases and identifies potential bugs
**Runs**: Scheduled before releases

---

### 📋 Code Reviewer
- **Role**: Code Quality & Standards
- **Responsibilities**:
  - Review code for quality and standards
  - Check for best practices
  - Ensure code is maintainable
  - Security review
  - Performance optimization suggestions

**Agent Task**: Performs thorough code reviews
**Runs**: Before major merges

---

## Team Workflow

1. **Development Phase**: Game Developer writes code
2. **Design Review**: UI/UX Designer reviews and provides feedback
3. **Testing Phase**: Tester creates test cases and identifies issues
4. **Code Review**: Reviewer checks quality and standards
5. **Leader Integration**: You review all feedback and make final decisions
6. **Implementation**: Updates based on team feedback

## Communication
- Each agent provides structured feedback
- Feedback is documented in separate files
- Leader coordinates and prioritizes changes
- All decisions logged for reference
